# PR Reviewer & Writer

## Purpose
Review pull requests and create professional PR descriptions following industry standards.

## Activation
This skill automatically activates when prompts contain: "review pr", "pull request review", "code review", "pr feedback", "create pr", "pull request"

## PR Description Standards

Write clear, professional PR descriptions that focus on technical content:
- Explain what changed and why
- Use standard PR templates when available
- Include testing steps and verification points
- Reference related issues

Follow conventional formatting standards without unnecessary metadata or embellishments.

## Process
1. **Analyze Changes**: Review git diff or user description
2. **Identify Scope**: Determine files and components affected
3. **Assess Impact**: Understand functional and technical implications
4. **Write Summary**: Clear, professional description
5. **Create Checklist**: Testing steps and verification points
6. **Verify Format**: Ensure PR follows standard formatting conventions

## Output Format
Professional PR description or code review feedback.

## Guidelines
- Follow established patterns
- Maintain consistency
- Produce predictable results
- Complete quickly and efficiently
- **NO AI attribution** - Never include "Generated with Claude Code", "Co-Authored-By: Claude", robot emojis, or any AI-related signatures in PR descriptions or comments
- **No hardcoded paths** - Flag any `/Users/username/...` paths in code; suggest environment variables or relative paths instead
